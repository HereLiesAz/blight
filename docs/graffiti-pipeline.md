# Graffiti Pipeline — Operator Guide

End-to-end overview of how addresses become graffiti scores, plus how to retrain, run inference standalone, and reuse the model elsewhere.

## Architecture

```
Kaggle dataset ──[Colab notebook]──▶ EfficientNet-B0 fine-tuned ──▶ models/model.onnx (~5 MB INT8)
                                              │
                                              └──▶ Google Drive (checkpoints, metrics, model card)

Sheet rows ──[GH Actions cron]──▶ scrape Street View tile ──▶ ONNX inference
                                                                    │
                                                                    ▼
                          writes graffiti_score, panoid, classified_at to Sheet
                                                                    │
                                                                    ▼
                                                         index.html: filter + rank
```

## Files

| Path | Purpose |
|---|---|
| `notebooks/train_graffiti_classifier.ipynb` | Colab fine-tune notebook |
| `notebooks/_build_notebook.py` | Programmatic builder for the notebook (regenerate after edits) |
| `models/model.onnx` | INT8-quantized model used at inference (gitignored, force-added) |
| `scripts/lib/streetview.py` | Throttled HTTP scraper of Street View panoramas + tiles |
| `scripts/lib/inference.py` | ONNX runtime wrapper |
| `scripts/lib/sheet.py` | Sheet column helpers |
| `scripts/streetview_scrape.py` | Standalone scraper CLI |
| `scripts/lib/enrichment.py` | NOLA cross-dataset lookups (zoning, footprint, case history, grass-cutting, land use) |
| `scripts/lib/drive_uploader.py` | Idempotent Google Drive thumbnail uploader |
| `scripts/enrich_properties.py` | Per-address enrichment job (invoked by GH Actions, between update and classify) |
| `scripts/classify_graffiti.py` | Per-address classification + thumbnail upload job (invoked by GH Actions) |
| `MODEL_CARD.md` | Model card (regenerated by the notebook each training run) |

## Drive thumbnail folder (one-time setup)

The classifier uploads a 256×256 thumbnail per property to a public Google Drive folder. Property cards in the map's list views render these inline.

**One-time:**
1. Create a folder in Google Drive (any name).
2. Share it with the service account email used by `GOOGLE_CREDENTIALS` (Editor permission).
3. Copy the folder ID from the URL: `https://drive.google.com/drive/folders/<FOLDER_ID>`.
4. Add it as a GitHub Actions secret named `STREETVIEW_DRIVE_FOLDER_ID`.
5. Ensure the GCP project has the Drive API enabled (alongside the Sheets API).

When the secret is missing, `classify_graffiti.py` skips the upload step and the rest of the pipeline keeps working — list cards just show "No street view image" placeholders.

## Enrichment job

`scripts/enrich_properties.py` runs between `update_database.py` and `classify_graffiti.py` in the nightly Actions cron. For each property missing zoning/land-use data, it queries five NOLA Socrata datasets:

| Field | Source |
|---|---|
| `zoning_class`, `zoning_desc` | Tabular Address Points (`cym7-cw5z`) — joined by `geopin` |
| `has_structure` | Building Footprint (`prh5-qsuf`) — joined by `geopin` |
| `case_count`, `earliest_case_date`, `days_under_blight` | Code Enforcement (`gjzc-adg8`) — same dataset, full case history per `geopin` |
| `last_grass_cut` | Grass Cutting / Lot Abatement (`xhih-vxs6`) — joined by `address` |
| `land_use_desc` | Future Land Use (`itxd-2247`) — point-in-polygon via `intersects(the_geom, 'POINT(<lng> <lat>)')` |

Idempotent: skips rows that already have zoning data. Throttled at 1 req/s by default (`ENRICH_MIN_INTERVAL_S`), capped at 100 properties per run (`ENRICH_MAX_PER_RUN`).

## Retraining

1. Open `notebooks/train_graffiti_classifier.ipynb` in Google Colab with a GPU runtime (T4 is sufficient).
2. Provide `kaggle.json` — preferred path is the Colab secret `KAGGLE_JSON`; the notebook also accepts a manual upload.
3. Mount Drive (Cell 2 prompts).
4. Runtime → Run all. ~10 min on T4.
5. Download `quantized.onnx` and `model_card.md` from `MyDrive/blight-graffiti/<run_id>/final/`.
6. In a local checkout:

   ```bash
   # Use the FP32 model.onnx (and its .data sidecar if present) for accuracy,
   # or the smaller INT8 quantized.onnx for faster CPU inference.
   cp <downloaded>/model.onnx models/model.onnx
   cp <downloaded>/model.onnx.data models/model.onnx.data 2>/dev/null || true
   cp <downloaded>/metrics.json models/metrics.json
   cp <downloaded>/model_card.md MODEL_CARD.md
   git add -f models/model.onnx
   git add models/model.onnx.data 2>/dev/null || true
   git add models/metrics.json MODEL_CARD.md
   git commit -m "Update graffiti classifier"
   git push origin main
   ```

   The `.gitignore` ignores `models/*.onnx`, so `git add -f` is required for the ONNX file. The `.onnx.data` sidecar (only present when the model exceeds protobuf's 2 GB limit, which can happen with the FP32 EfficientNet-B0 + external-data export) is NOT gitignored and adds normally — it must sit alongside the `.onnx` file for ONNX Runtime to load.

The next scheduled `Update Database` workflow run will pick up the new model and start scoring rows.

### Editing the notebook

The `.ipynb` file is generated by `notebooks/_build_notebook.py`. To change a cell, edit the corresponding constant in the builder script, then run:

```bash
python notebooks/_build_notebook.py
```

Cell IDs are stable (`cell-01-intro`, `cell-02-setup`, …) so the regenerated `.ipynb` produces a clean diff.

## Running inference standalone (no Sheet)

```bash
# Fetch one Street View tile to disk
python -m scripts.streetview_scrape --lat 29.964 --lng -90.007 --out cache/sample.jpg

# Score it
python -c "
from scripts.lib.inference import GraffitiClassifier
print(GraffitiClassifier('models/model.onnx').score(open('cache/sample.jpg','rb').read()))
"
```

## Running the full classification job locally

```bash
export GOOGLE_CREDENTIALS="$(cat path/to/service-account.json)"
export MODEL_PATH=models/model.onnx
export GRAFFITI_MAX_PER_RUN=10        # cap for testing
export GRAFFITI_MIN_INTERVAL_S=4.0    # be kind to the source IP
python -m scripts.classify_graffiti
```

Without `MODEL_PATH` (or with a path that does not exist), the job exits 0 with a `Model not found ... skipping classification.` message. This is intentional so the GH Actions workflow does not fail before the first model is trained.

## Reusing the model in another project

**Python (any platform with ONNX Runtime):**

```python
from scripts.lib.inference import GraffitiClassifier
clf = GraffitiClassifier('models/model.onnx')
score = clf.score(open('photo.jpg', 'rb').read())   # 0.0–1.0
```

**Browser / JavaScript** — use [`onnxruntime-web`](https://github.com/microsoft/onnxruntime/tree/main/js/web). Apply the same preprocessing as `scripts/lib/inference.py::_preprocess`:

1. Decode JPEG → RGB
2. Resize so the shorter side is 256 px (preserve aspect ratio)
3. Center-crop 224×224
4. Divide by 255.0
5. Subtract ImageNet mean `[0.485, 0.456, 0.406]`, divide by ImageNet std `[0.229, 0.224, 0.225]`
6. Transpose to NCHW (1, 3, 224, 224), float32
7. Apply sigmoid to the model's single logit output → probability of graffiti

**REST microservice** — wrap `GraffitiClassifier` in FastAPI; runs on a small CPU box or AWS Lambda.

## Scraper failure modes

| Symptom | Cause | Fix |
|---|---|---|
| Persistent HTTP 429 | Source IP rate-limited | Increase `GRAFFITI_MIN_INTERVAL_S`; rotate runner |
| `PanoramaNotFound` for many addresses | No Street View coverage at lat/lng | Acceptable; row gets `graffiti_panoid="NO_PANO"` and `score=0.0` |
| Body parse error in `lookup_panoid` | Google changed the `SingleImageSearch` response shape | Capture a fresh response, update `_strip_xss_prefix` / index path in `scripts/lib/streetview.py`, regenerate `tests/fixtures/photometa_response.txt` |
| 200 OK but body is HTML CAPTCHA | IP flagged | Stop the runner; switch network. The pipeline cannot defeat reCAPTCHA |

## Domain-shift sanity check

The model is trained on Kaggle photos, not Street View. Before relying on it for ranking decisions, run the optional last cell of the notebook on a hand-labeled set of ~50 Street View crops from real Lower Ninth Ward addresses. If AUC < 0.7, treat scores as a weak signal only.

## Privacy

The pipeline runs server-side (GitHub Actions); no end-user telemetry is involved. The scraped imagery cache is not persisted between runs unless explicitly enabled.
